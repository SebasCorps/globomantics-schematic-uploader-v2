# Security Policy

## Supported Versions

We release patches for security vulnerabilities as needed. Please ensure you are using the latest version.

## Reporting a Vulnerability

If you discover a security vulnerability, please email timothy@timothywarner.org. Do **not** create a public GitHub issue.

We will respond as quickly as possible and coordinate a fix and disclosure.

## General Guidelines
- Keep dependencies up to date.
- Never commit secrets or credentials.
- Use GitHub security features (code scanning, Dependabot, secret scanning).

For more, see [GitHub Security Best Practices](https://docs.github.com/en/code-security/getting-started/securing-your-repository).
